# 个人待办提醒助手

一个适合个人使用的待办提醒小网页。它可以随手记录事情、简单识别中文时间、提前提醒、标记完成，并支持给一条事情附一张手绘草图。

当前版本：`0.5.1`

## 适合用来做什么

- 记录临时想法和待办
- 设置“今天15:00做某事，提前2分钟提醒”
- 查看今天、本周、月历里的待办
- 备份和恢复本地数据
- 安装成 PWA，支持离线打开
- 支持中文 / English 界面切换
- 编辑已有任务、时间、标签和重复规则
- 搜索内容或标签
- 把没有明确时间的事情先放到“以后”
- 简单记录支出
- 做临时四则运算
- 在电脑或手机浏览器里快速试用

## 电脑上打开

如果只是想先测试，可以直接双击：

```text
public/index.html
```

这种方式会把数据保存在当前浏览器里，适合先试感觉。

## 本地服务模式

如果想让数据保存到本地文件，或者让手机在同一个 Wi-Fi 下访问同一份列表，需要运行本地服务：

```bash
npm start
```

然后打开：

```text
http://127.0.0.1:4173
```

这个地址只给当前电脑自己访问。

## 手机访问同一份列表

电脑和手机连接同一个 Wi-Fi 后，在电脑上运行：

```bash
HOST=0.0.0.0 npm start
```

然后查看电脑的局域网 IP，例如 `192.168.1.23`，手机浏览器打开：

```text
http://192.168.1.23:4173
```

手机和电脑看到的是同一个数据文件。

不要在公司、酒店、咖啡店等公共网络里随便开启这个模式。当前版本没有登录密码，同一个局域网里知道地址的人可能访问你的待办。

## GitHub Pages 分享

这个项目可以直接发布到 GitHub Pages。别人访问 GitHub Pages 链接时，使用的是纯前端模式：

- 每个人的数据只保存在自己的浏览器里
- 别人看不到你的本地待办
- 你也看不到别人的待办
- GitHub Pages 不会运行 `server.js`

发布时可以选择仓库根目录作为 Pages 来源。根目录的 `index.html` 会自动打开 `public/index.html`。

## 数据保存位置

直接打开 `public/index.html` 或访问 GitHub Pages 时，数据保存在浏览器本地存储里。

当前版本会优先使用浏览器自带的本地数据库保存数据，并保留一份本地存储副本。这样仍然不收费，也不会上传到外部服务器。

## PWA 安装和离线

用本地服务或 GitHub Pages 打开时，浏览器会识别它为可安装网页：

- 电脑上通常可以在地址栏或浏览器菜单里安装
- 手机上可以用浏览器的“添加到主屏幕”
- 安装后像 App 一样从桌面打开
- 已访问过的页面文件可以离线打开
- 数据仍然保存在当前设备和当前浏览器里

直接双击 `public/index.html` 时，网页功能可以用，但浏览器不会启用 PWA 安装和离线缓存。

运行本地服务时，待办会保存在：

```text
data/tasks.json
```

`data/` 已经写进 `.gitignore`，避免误传真实待办。

- 快速新增事情

## 怎么测试提醒

1. 打开页面后，先点“开启声音和弹窗”。按钮会变成“关闭声音和弹窗”。
2. 输入一条离现在几分钟后的事情，例如：

```text
今天15:30测试提醒 提前2分钟提醒
```

3. 保持网页打开。到 15:28 左右，它会在页面顶部显示提醒，并响一声。

如果没有写“提前多久”，默认提前 2 分钟提醒。

## 怎么测试重复提醒

可以在输入框选择“每周”，也可以直接写：

```text
今天16:00看股票 提前2分钟提醒 每天
```

提醒触发后，这条事项会保留提醒历史。如果设置了重复，它会自动把下一次提醒时间往后推。

当前版本的提醒依然要求网页保持打开。

## 怎么测试 0.3 时间识别

可以分别输入：

```text
半小时后提醒我喝水
下周一上午10点开会
每周三下午3点看股票
每个月1号交房租
工作日早上9点看计划
```

带日期的事项会出现在日历里。点日历上的某一天，可以只看那天的待办。

提醒弹出后，可以直接选择“完成”“延后10分钟”“明天再提醒”或“跳过本次”。

## 怎么记账

切换到“记账”，填写金额和分类，例如：

```text
买菜
```

金额填 `38.5`，分类填 `买菜`。页面上会显示本月支出合计。

## 安全说明

- 不要把银行卡密码、证券账户密码、身份证完整号码等敏感信息写进待办。
- 不要把 `data/tasks.json` 上传到 GitHub。
- `127.0.0.1` 只代表你自己的电脑，朋友打开这个地址不会访问到你的电脑。
- `HOST=0.0.0.0 npm start` 会让同一局域网里的设备有机会访问这个网页，建议只在可信网络里使用。
